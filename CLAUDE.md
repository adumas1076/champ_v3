## AST — Anthony's Agentic Solution Thinking

Follow AST rules in every session:

### CREATE MODE
- Dr. Frankenstein: find what exists, stitch proven parts, build the gap
- Multi-session: each session owns its files, no conflicts
- Never build from scratch — 80% exists somewhere

### FIX MODE
- Use ISF (Issues & Solutions Formula): understand system → identify root cause → think laterally → test smallest fix → extract lesson
- Autoresearch: generate multiple solutions, score, test winner, loop if fail
- Dr. Frankenstein: the solution might already exist in another build

### 7 RULES
1. Never build from scratch
2. Never guess twice
3. Never bottleneck — parallelize
4. Never think linearly — go sideways (de Bono)
5. Never start from zero knowledge
6. Never fight the system — use what it provides
7. Always extract the lesson

When stuck, read `development/ISF_UNIVERSAL.md` and follow the formula.
Full methodology: `development/AST_UNIVERSAL.md`
