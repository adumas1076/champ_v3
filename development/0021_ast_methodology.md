## One-Line Definition

> **AST is the methodology for creating anything by stitching proven parts, diagnosing issues with first principles, solving with self-improving loops, thinking laterally, and executing in parallel — never from scratch, never guessing twice, never bottlenecked.**

---

## Two Modes

### Mode 1: CREATE (Building something new)

```
INPUT: "I want to build X"
    → QUERY: What exists that I can stitch together?
    → DR. FRANKENSTEIN: Find working parts from references, frameworks, repos, specialists
    → MULTI-SESSION: Split build across parallel sessions (no conflict rule)
    → OUTPUT: Goal achieved — built from proven parts, not scratch
```

### Mode 2: FIX (Something broke)

```
INPUT: "X isn't working. Y is happening."
    → ISF: Diagnose correctly (understand system → identify root cause)
    → LATERAL THINKING: Think sideways — where is this already solved?
    → AUTORESEARCH: Generate candidates → score → test → keep/discard → loop
    → DR. FRANKENSTEIN: Solution might exist in another build — stitch it in
    → MULTI-SESSION: Parallelize the fix across sessions
    → OUTPUT: Issue solved + lesson extracted + system improved
```

## AST Rules

1. **Never build from scratch.** 80% of any build already exists somewhere. Find it, stitch it, build the 20% gap.
2. **Never guess twice.** If a solution fails, you have new data. Use it. ISF loops with information, not hope.
3. **Never bottleneck.** Split work across parallel sessions. Each owns its files. No conflicts.
4. **Never think linearly.** Solutions come from any domain. Frameworks, ideologies, industries, principles. de Bono sideways.
5. **Never start from zero knowledge.** Business Matrix has proven frameworks. Reference builds have working code. Past ISF lessons have solved patterns.
6. **Never fight the system.** If your solution requires hacking around the system's design, you're thinking wrong. Use what the system provides.
7. **Always extract the lesson.** Every build, every fix, every session produces a reusable pattern. Store it. Next time starts better.
